# Conventions — Соглашения по оформлению

## Общие правила

### Форматирование

- Markdown с кодировкой UTF-8
- Мягкая переносная ширина строк: 120 символов
- Отступы: пробелы (2 пробела для вложенности)
- Заголовки: иерархия `#` → `##` → `###` → `####`
- Списки: дефисы `-` для неупорядоченных, цифры `1.` для упорядоченных

### Стиль текста

- Названия секций — в именительном падеже
- Команды и код — в `моноширинном` формате
- Важные термины — **жирным** при первом упоминании
- Ссылки — с относительными путями

### Системный промпт (prompt.md)

#### Структура

```markdown
# <ROLE NAME>

## 1. Identity & Purpose
## 2. Context & Domain
## 3. Decision Framework
## 4. Interaction Rules
## 5. Output Format
## 6. Anti-Patterns
## 7. Quick Reference
```

#### Правила оформления

- Каждый раздел — **независимая единица знаний** (self-contained)
- Приоритеты обозначаются цифрами: `🔴 must`, `🟡 should`, `🟢 can`
- Правила в одном разделе не должны противоречить правилам в другом
- Используйте нумерованные списки для последовательных шагов
- Используйте маркированные списки для перечислений

#### Пример приоритетов

```markdown
### Приоритеты

1. **🔴 Must** — безоговорочные правила, нарушение = провал контракта
2. **🟡 Should** — сильные рекомендации, отклонение с обоснованием
3. **🟢 Can** — опциональные улучшения
```

### Карточка (card.md)

#### Структура

```markdown
# <ROLE NAME> — Card

## Metadata
## Description
## Input / Output
## Scope & Boundaries
## Constraints & Anti-Patterns
## Usage Examples
## Validation Status
## Related Files
```

#### Metadata

```markdown
| Field      | Value                    |
|------------|--------------------------|
| Name       | python-architect         |
| Version    | v1.0.0                   |
| Author     | <author>                 |
| Status     | validated                |
| Created    | 2026-06-21               |
| Updated    | 2026-06-21               |
| Category   | architecture             |
```

#### Статусы

| Статус      | Значение                                  |
|-------------|-------------------------------------------|
| `draft`     | Черновик, не валидирован                  |
| `testing`   | Проходит тестирование                     |
| `validated` | Прошёл все тесты, готов к использованию   |
| `deprecated`| Устарел, заменяется другой версией         |

### Тестовые кейсы (test-cases.md)

#### Структура

```markdown
# <ROLE NAME> — Test Cases

## Test Suite Overview
## Test Cases
## Edge Cases
## Metrics
```

#### Формат тест-кейса

```markdown
### TC-001: <Краткое описание>

- **Input:** <входные данные>
- **Expected:** <ожидаемый результат>
- **Criteria:** <критерий оценки>
- **Status:** ✅ / ❌ / ⏳
```

### Changelog (changelog.md)

#### Структура

```markdown
# <ROLE NAME> — Changelog

## [v1.0.0] — 2026-06-21

| Field    | Value              |
|----------|--------------------|
| Version  | v1.0.0             |
| Date     | 2026-06-21         |
| Author   | <author>           |
| Type     | feat               |
| Summary  | Initial version    |

### Changes
- ...
```

#### Типы изменений (Semantic Versioning)

| Тип     | Версия      | Описание                        |
|---------|-------------|---------------------------------|
| `feat`  | MAJOR       | Новое поведение / добавление    |
| `fix`   | PATCH       | Исправление без нового поведения|
| `refactor` | PATCH    | Изменение формы без смены поведения |
| `docs`  | PATCH       | Обновление документации         |
| `test`  | PATCH       | Обновление тестов              |
| `chore` | PATCH       | Рутинные изменения             |

### Метаданные проекта

| Поле         | Значение              |
|--------------|-----------------------|
| Версия либы  | v1.0.0                |
| Формат       | Markdown (GFM)        |
| Кодировка    | UTF-8                 |
| EOL          | LF                    |
| Max line     | 120 chars             |
